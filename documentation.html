About Page Replacement Algorithms

Page replacement algorithms are used by operating systems to decide which memory pages to swap out when new pages need to be allocated. This visualizer helps you understand how different algorithms work.

Available Algorithms

FIFO (First-In-First-Out)

Replaces the oldest page in memory. Simple to implement but may not be efficient as it doesn't consider usage patterns.

LRU (Least Recently Used)

Replaces the page that hasn't been used for the longest time. Better performance than FIFO but requires more overhead to track usage.

Optimal

Replaces the page that won't be used for the longest time in the future. Provides the theoretical minimum page faults but not practical as it requires future knowledge.

LFU (Least Frequently Used)

Replaces the page that has been used the least often. Good for some workloads but may not adapt quickly to changing access patterns.

MRU (Most Recently Used)

Replaces the most recently used page. Works well in some specific scenarios where older pages are more likely to be used again.

Second Chance

An enhancement to FIFO that gives pages a "second chance" before replacement. Pages are marked with a reference bit that is checked before replacement.

Clock (Approximation)

A practical implementation of Second Chance that uses a circular buffer (like a clock face) to track pages and their reference bits.

How to Use

  1. Select an algorithm from the dropdown
  2. Set the number of frames (memory capacity)
  3. Enter a reference string or generate one with a specific pattern
  4. Click "Visualize" to see the algorithm in action
  5. Use the step buttons to go through each reference
  6. Compare algorithms to see which performs best for your reference string
  7. Generate statistics to see performance across different frame counts
